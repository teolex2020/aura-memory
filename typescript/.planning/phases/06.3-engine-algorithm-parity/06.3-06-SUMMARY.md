## Task Summary

### Task 1: Cluster Guards — Tag Barrier, Family Guard, Semantic Type Bridge, Generic Family (TDD)

**RED commit:** `dc652cf` — 7 failing tests for guard blocking behavior (stubs always allow).
**GREEN commit:** `1e32d9f` — All 4 guards implemented, 93 tests pass.

**What was done:**

1. **parseBeliefKeyFamily** — Extracts family from belief key: `namespace:tags:semantic_type` → returns `parts[1]` (the tags portion). Matching Rust `parse_belief_key_family` (concept.rs:1357).

2. **familyTokenSet** — Splits comma-separated family string into lowercase token set. Extracted from Rust `family_token_set` (concept.rs:1366).

3. **isGenericFamily** — Checks `family === "alerts"`. Extracted from Rust `is_generic_family` (concept.rs:1374).

4. **tagBarrier** — Blocks merge when `shared_tags == 0` AND both beliefs have tags. Matching Rust lines 1038-1042/918-923.

5. **familyGuard** — Returns `"allowed"` (same family), `"bridge_allowed"` (overlap bridge or single-tag fact↔decision bridge), or `"blocked"`. Matching Rust lines 1044-1078:
   - **Overlap bridge:** `fam_i.len() >= 2 && fam_j.len() >= 2 && family_overlap >= 1 && shared >= 1 && !is_generic_family(...)` 
   - **Single-tag bridge:** `union_mode == SingleTagFactDecisionBridge && fam_i.len() == 1 && fam_j.len() == 1 && shared >= 2 && !is_generic_family(...) && fact↔decision crossing`
   - Critical: bridge checks only apply when families DIFFER (`family_i != family_j` in Rust)

6. **genericFamilyGuard** — `"alerts"` family requires `shared_tags >= 2`. Matching Rust lines 1080-1082/961-963.

7. **semanticTypeBridge** — Same semantic_type → ALLOWED. Different but overlapping tags → ALLOWED. Otherwise → BLOCKED.

8. **clusterBeliefs integration** — Both `clusterBeliefs` (SdrTanimoto) and `clusterBeliefsCanonical` (CanonicalFeature) now apply the 4-layer guard chain before similarity computation. `discover()` builds `belief_families` and `belief_semantic_types` maps.

### Task 2: Surface.ts Rewrite — Per-ns Cap, Dedup, 5-dim Tiebreak, Candidate Threshold 0.70 (TDD)

**RED commit:** `a84f455` — 19 failing tests (stub returns empty).
**GREEN commit:** `f1a6e26` — `computeSurfaceConcepts` implemented, all 149 tests pass.

**What was done:**

1. **computeSurfaceConcepts(concepts, namespaces?)** — Pure sync function matching Rust `surface_concepts_filtered()`:
   - **Filter:** Stable always passes; Candidate needs `score >= 0.70`; Rejected excluded; provenance check (non-empty belief_ids/record_ids); content check (non-empty core_terms or tags)
   - **Sort:** 5-dim tiebreak (abstraction_score DESC → confidence DESC → cluster_size DESC → Stable before Candidate → key ASC)
   - **Collect:** Dedup by key FIRST, then per-ns cap (verified against Rust lines 1924-1939)
   - **Constants:** `MAX_SURFACED_PER_NAMESPACE = 5`, `SURFACE_CANDIDATE_THRESHOLD = 0.70`

2. **Backward compat** — `surfaceConcepts(engine, limit?)` and `surfaceConceptsFiltered(engine, limit?, namespace?)` are Effect wrappers that call `computeSurfaceConcepts` internally. EpistemicRuntime imports work unchanged.

3. **Test infrastructure** — `c()` helper updated with non-empty defaults (`belief_ids: ["b-default"]`, `record_ids: ["r-default"]`, `core_terms: ["term-default"]`) to pass Rust provenance filter.
