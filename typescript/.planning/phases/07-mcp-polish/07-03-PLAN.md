- Lock search ownership explicitly:
  - `Aura` must own a normalized searchable record view in memory
  - that view is populated from the same cognitive-record load/normalize path used by maintenance
  - write operations (`store` / `update` / `delete` / `connect` where relevant) must refresh the same in-memory view
  - `search` implements the concrete Rust semantics over that view: namespace defaulting, optional query substring match, `level`/`tags`/`content_type`/`source_type`/`semantic_type` filters, then Rust-aligned importance ordering and limit truncation
- Pick and document one concrete search-view mutation strategy in `07-03-SUMMARY.md` before coding the write refresh path:

  - acceptable: a dedicated mutable `Ref` if the write/update path stays coherent and test-covered
  - forbidden: implicit mutation of multiple partially overlapping caches

- Align `insights` to the surface the Rust MCP server actually calls. Use `../src/mcp.rs` as the source of truth, decide whether TS exposes `stats()`, `insights()`, or both, and record the naming/output contract in `07-03-SUMMARY.md` so `07-07` does not rediscover it while wiring handlers.
