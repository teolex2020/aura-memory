<objective>
Convert all inline `import("xxx").A` type expressions in the 6 `contract` source files to proper top-level `import type` statements, and replace each inline occurrence with the short name.

Purpose: Eliminate dynamic import type expressions for consistency, readability, and TypeScript best practices. The existing `FileRead.ts` and `FileWrite.ts` already use the proper `import type { Effect } from "effect"` pattern — this change brings the contract type definitions in line with that convention.

Output: 6 files refactored, 0 inline `import(...)` type expressions remaining, code compiles cleanly.

<context>
The 6 files being modified are all in `packages/contract/src/` and contain type definitions for the Aura contract layer. The target pattern is:

```typescript
// BEFORE
type Foo = {
  bar: () => import("effect").Effect.Effect<void>
  baz: () => import("./FileRead").FileRead
}

// AFTER
import type { Effect } from "effect"
import type { FileRead } from "./FileRead"

type Foo = {
  bar: () => Effect.Effect<void>
  baz: () => FileRead
}
```

Key facts:
- `import("effect").Effect.Effect` appears in all 6 files
- `import("./FileRead").FileRead` appears in Belief.ts, Causal.ts, Concept.ts, Policy.ts
- `import("./FileWrite").FileWrite` appears in Belief.ts, Causal.ts, Concept.ts, Policy.ts
- `import("./EpistemicTrace").EpistemicTrace` appears only in Belief.ts (2 occurrences)
- Concept.ts already has `import type { EpistemicTrace } from "./EpistemicTrace"` — no duplicate needed
- EpistemicRuntime.ts and Recall.ts only need the `Effect` import

Existing import style in these files is consistent: regular imports via `import { X } from "..."`, type-only imports via `import type { X } from "..."`. New imports should follow the same pattern and be inserted after the last existing import line, grouped by module origin.
</context>
