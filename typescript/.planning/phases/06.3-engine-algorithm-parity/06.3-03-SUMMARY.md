## Task Summary

### Task 1: Implement 4 SDR subcluster guard functions + coarse key mode alignment (TDD)

**RED commit:** `006e7d7` — 16 failing tests for subcluster functions, coarse key modes, dispatch constants.

**GREEN commit:** `3c701e1` — All 4 subcluster functions implemented, dispatch fixed, threshold corrected.

**What was done:**

1. **sdrSubcluster (plain):** Exported alias for existing `unionFindClusters`, threshold-parameterized. Mirrors Rust belief.rs:678-733.

2. **sdrSubclusterTagGuarded:** Tag barrier requiring shared_tags >= 1 before Tanimoto check. Exception: if either side has no tags, skip the barrier (matching Rust belief.rs:781). Used by DualKey and NeighborhoodPool modes.

3. **sdrSubclusterBridgeGuarded:** Bridge tag overlap guard. Normalizes each record's tags via `normalizeBridgeTag` (ui→frontend, auth→authentication, deploy→release), then requires at least one shared normalized bridge tag before Tanimoto check. Mirrors Rust belief.rs:810-875.

4. **sdrSubclusterTagSdrGuarded:** Dual guard (tag fingerprint + content SDR). Generates tag SDR fingerprints via `canonicalTagText` + `SDRInterpreter.textToSdrLowered`, requires tag fingerprint Tanimoto >= 0.08 before content SDR comparison. Mirrors Rust belief.rs:882-957.

5. **Threshold fix:** `NEIGHBORHOOD_POOL_THRESHOLD = 0.08` (was incorrectly 0.10). Verified from Rust belief.rs:1045 where NeighborhoodPool uses `unwrap_or(0.08)`.

6. **Dispatch fix:** `update_with_sdr` now correctly dispatches to the appropriate subcluster function per `coarse_key_mode` instead of using `unionFindClusters` for all modes.

7. **Helper functions:** `normalizeBridgeTag`, `normalizedBridgeTags`, `canonicalTagText` matching Rust implementations.

8. **TagSdrGenerator interface:** Decoupled SDR fingerprint generation from SDRInterpreter dependency for testability.
