    **applySuppression function:**
    `(hints: PolicyHint[]) => PolicyHint[]`

    Algorithm (Rust policy.rs:727-890):
    1. Group hints by namespace+domain key
    2. For each group, check each pair:
       a. Same namespace (already grouped by it)
       b. Same domain (already grouped by it)
       c. Opposite polarity (one Positive/Prefer/Recommend vs one Negative/Avoid/VerifyFirst)
       d. Overlapping cause_record_ids (hint.effect_keys or internal cause_record_ids overlap — standardize on cause_record_ids matching Rust)
    3. If conflict detected:
       - Compare policy_strength
       - Suppress the lower-strength hint (set state to Suppressed)
    4. Return hints with updated states

    **Suppression rules:**
    - Only suppress if both hints have state === "Stable" or "Candidate"
    - Do not suppress already-suppressed hints
    - If policy_strengths are equal (within 0.01): suppress neither (conservative)
    - Suppressed hint: state = "Suppressed"

    **Surface.ts rewrite (surfacePolicyHints):**
    `(hints: ReadonlyArray<PolicyHint>, domains?: ReadonlyArray<string>) => ReadonlyArray<SurfacedPolicyHint>`

    Algorithm:
    1. Filter:
       - provenance check: hint has valid pattern_id
       - domain non-empty: hint.domain.length > 0
       - recommendation non-empty: hint.recommendation.length > 0
       - Keep only candidate/stable state hints (not Suppressed, not Rejected)
    2. Apply candidate threshold:
       - policy_strength >= POLICY_STRENGTH_THRESHOLD (0.70) — verify exact Rust threshold
       - AND confidence >= CONFIDENCE_THRESHOLD (0.55) — verify exact Rust threshold
    3. Sort (5-dim tiebreak):
       a. policy_strength DESC (primary)
       b. confidence DESC
       c. risk_score DESC (higher risk surfaced first)
       d. Stable before Candidate (stable priority)
       e. key ASC (deterministic tiebreaker)
    4. Group by domain
    5. Per domain: apply cap MAX_SURFACED_PER_DOMAIN = 3 (top 3 sorted) — verify exact Rust cap
    6. Return flat array

    **SurfacedPolicyHint type:**
    - id, namespace, domain, action_kind, polarity, condition, recommendation
    - policy_strength, confidence, utility_score, risk_score

    Constants (in Surface.ts):
    - POLICY_STRENGTH_THRESHOLD = 0.70
    - CONFIDENCE_THRESHOLD = 0.55
    - MAX_SURFACED_PER_DOMAIN = 3

    **Integration into discover:**
    After all hints built and scored:
    1. For each hint: if policyStrength >= Rust candidate threshold → state = "Stable"; else state = "Candidate"
