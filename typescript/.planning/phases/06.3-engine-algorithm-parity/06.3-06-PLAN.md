    Per D-14: Implement 4-layer cluster guard system in ConceptEngine.clusterBeliefs (or equivalent).

    Add to ConceptEngine the following guard functions:

    **tagBarrier(beliefA, beliefB, sdrLookup): boolean**
    - Check shared_tags between the two beliefs' prototype records
    - Collect all unique tags from both beliefs' records
    - Compute intersection count
    - If shared_tags == 0: return BLOCKED (except if either side has no tags)
    - If shared_tags >= 1: return ALLOWED

    **familyGuard(beliefA, beliefB): 'blocked' | 'bridge_allowed' | 'allowed'**
    - Extract family tags from both beliefs (narrowest non-"alerts" tag, or first tag)
    - If same family → ALLOWED
    - If different families:
      - Check overlap bridge: token overlap between families AND shared_tag exists → bridge_allowed
      - Check union_mode === SingleTagFactDecisionBridge: fact vs decision semantic_type → bridge_allowed
      - Otherwise → BLOCKED

    **genericFamilyGuard(family: string, sharedTags: number): boolean**
    - If family === "alerts": require shared_tags >= 2
    - Otherwise: require shared_tags >= 1

    **semanticTypeBridge(beliefA, beliefB): boolean**
    - If same semantic_type → ALLOWED
    - If different but tags overlap → ALLOWED (bridge via shared meaning)
    - Otherwise → BLOCKED (can merge across semantic types only if tags align)

    **clusterBeliefs integration:**
    When merging two belief clusters via Union-Find, apply guards in order:
    1. tagBarrier: if BLOCKED → skip pair (don't even check Tanimoto)
    2. familyGuard: if BLOCKED → skip pair
    3. genericFamilyGuard: if BLOCKED → skip pair
    4. semanticTypeBridge: if BLOCKED → skip pair
    5. ONLY THEN: check Tanimoto (or Jaccard) similarity threshold

    Write tests before implementation (TDD RED phase).
  </action>

  <action>
    Per D-15: Rewrite Surface.ts surface function to match Rust.

    **surfaceConcepts function:**
    `(concepts: ReadonlyArray<ConceptCandidate>, namespaces?: ReadonlyArray<string>) => ReadonlyArray<SurfacedConcept>`

    Algorithm:
    1. Filter: keep concepts where abstraction_score >= SURFACE_CANDIDATE_THRESHOLD (0.70)
    2. Filter: remove concepts where key is empty or namespace is empty
    3. Sort (5-dim tiebreak):
       a. abstraction_score DESC (primary)
       b. confidence DESC (secondary)
       c. cluster_size (belief_ids.length) DESC
       d. Stable before Candidate (stable priority)
       e. key ASC (deterministic alphabetical tiebreaker)
    4. Group by namespace
    5. Per namespace: apply cap MAX_SURFACED_PER_NAMESPACE = 5 (take top 5 sorted)
    6. Dedup by key within results (first occurrence wins)
    7. Return flat array of SurfacedConcept

    **SurfacedConcept type** (import from contract or define locally):
    - id: string
    - key: string
    - namespace: string
    - abstraction_score: number
    - confidence: number
    - core_terms: ReadonlyArray<string>
    - shell_terms: ReadonlyArray<string>
    - state: string

    Constants:
    - MAX_SURFACED_PER_NAMESPACE = 5
    - SURFACE_CANDIDATE_THRESHOLD = 0.70

    Write tests before implementation (TDD RED phase).

    Note: If namespaces parameter is provided, only include concepts from those namespaces. If omitted, include all.
  </action>
