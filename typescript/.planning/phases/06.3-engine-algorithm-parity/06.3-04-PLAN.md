    Per D-03 and AUDIT-DIFF 1.7: Rewrite apply_layer_feedback to match Rust's algorithm.

    New signature (uses Interface types per project layering rules — NOT CausalEngineImpl/PolicyEngineImpl):
    `(causalEngine: CausalEngine.Interface, policyEngine: PolicyEngine.Interface) => Effect<BeliefFeedbackReport, never, EpistemicTrace>`
    
    Note: This requires importing `CausalEngine` and `PolicyEngine` from `@aura/contract`. The contract types imported in Plan 01 define their Interface types. If CausalEngine.Interface does not expose a `stats()` method (it may expose via a Tag effect), use the available API — the goal is to iterate over causal patterns and policy hints.

    DIFFERENCE FROM RUST: Rust's apply_layer_feedback receives `&CausalEngine` and `&PolicyEngine` and directly reads their states. In TS effect-smol, we read state via `causalEngine.stats()`. This is an acceptable adaptation: same data source, effect-smol compatible API.

    BeliefFeedbackReport type — check if Plan 01 added it to BeliefTypes.ts. If not, define locally in BeliefEngine.ts with the Rust-matching fields. Add it to Plan 01 closure if discovered missing.

    Algorithm (mirroring Rust belief.rs:1287-1350):
    1. Get causal engine state (stats()) → iterate patterns
       - For each stable pattern with causal_strength > 0.5: positive confidence_delta
       - For each pattern where cause_belief_id matches a known belief_id
    2. Get policy engine state (stats()) → iterate hints
       - For each stable hint with policy strength > 0.55: positive confidence_delta if Prefer/Recommend
       - For Avoid/VerifyFirst hints: negative confidence_delta
       - For Suppressed hints: no contribution
    3. Aggregate per belief_id:
       - Sum confidence_deltas from all signals
       - Sum volatility_deltas from contradictory signals (conflicting polarity on same belief)
    4. Apply bounded clamping:
       - Clamp confidence_delta: Math.max(-0.18, Math.min(0.08, delta))
       - Clamp volatility_delta: Math.max(-0.06, Math.min(0.20, delta))
       - Verify clamp bounds against Rust source (grep for clamping values in belief.rs feedback function)
       - Update belief.confidence += confidence_delta
       - Update belief.volatility += volatility_delta
    5. Do NOT modify stability (stability is ONLY changed by resolve())
    6. Build BeliefFeedbackReport with per-belief entries

    BeliefFeedbackReport type:
    - beliefsTouched: number
    - beliefsBoosted: number
    - beliefsDampened: number
    - netConfidenceDelta: number
    - netVolatilityDelta: number
    - entries: ReadonlyArray<FeedbackAuditEntry>

    Each FeedbackAuditEntry:
    - beliefId, sourceKind, sourceId, reason
    - deltaRequested (before clamp), deltaApplied (after clamp)
    - confidenceBefore, confidenceAfter
    - volatilityBefore, volatilityAfter
    - stabilityBefore, stabilityAfter (both unchanged — for completeness)
