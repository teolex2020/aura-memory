<feature>
  <name>Policy surface function: filter, sort, limit, map disclosed policy hints</name>
  <files>
    packages/policy/src/Surface.ts (NEW)
    packages/policy/src/Surface.test.ts (NEW)
  </files>
  <behavior>
    surfacePolicyHints(engine, limit?):
      - Reads engine state via engine.stats()
      - Filters hints where state is PolicyState.Stable or PolicyState.Candidate
      - Sorts by policyStrength descending, then by actionKind priority (Avoid > Warn > VerifyFirst > Recommend > Prefer) for equal strength
      - Caps at limit (default 20, no upper cap)
      - Maps each PolicyHint to SurfacedPolicyHint with camelCase fields
      - Returns ReadonlyArray&lt;SurfacedPolicyHint&gt;
      - Empty state returns []

    surfacePolicyHintsFiltered(engine, limit?, namespace?):
      - Same as surfacePolicyHints but additionally filters by .namespace === namespace
      - If namespace not provided: same as surfacePolicyHints

    Action kind priority mapping:
      "avoid" -> 5 (highest priority)
      "warn" -> 4
      "verify_first" -> 3
      "recommend" -> 2
      "prefer" -> 1 (lowest priority)

    Test cases:
      1. surfacePolicyHints returns eligible hints sorted by policyStrength desc
      2. surfacePolicyHints respects limit
      3. surfacePolicyHintsFiltered filters by namespace
      4. surfacePolicyHints on empty engine returns []
      5. surfacePolicyHintsFiltered with non-existent namespace returns []
      6. surfacePolicyHints excludes Suppressed/Rejected state hints
      7. Equal policyStrength uses actionKind priority tiebreaker (Avoid before Prefer)
      8. surfacePolicyHints default limit is 20
  </behavior>
  <implementation>
    Per D-11, surface functions are implemented in @aura/policy (not in EpistemicRuntime). Per D-17, TDD tests verify correctness.

    Import { Effect } from "effect"
    Import { SurfacedPolicyHint, PolicyActionKind } from "@aura/contract"
    Import type { PolicyEngineImpl, PolicyEngineState, PolicyHint } from "@aura/contract"

    The action kind priority map is a const Record<string, number> at module level.
    The sort comparator uses both policyStrength desc and actionKindPriority desc.

    Implementation steps:
    - yield* engine.stats()
    - Object.values(state.hints)
    - Filter: hint.state === PolicyState.Stable || hint.state === PolicyState.Candidate
    - Sort: hint.policyStrength desc → actionKindPriority desc → hint.id asc (deterministic)
    - Slice to max = limit ?? 20
    - Map each hint to SurfacedPolicyHint:
      - state: hint.state (string)
      - actionKind: hint.actionKind
      - namespace: hint.namespace
      - domain: hint.domain
      - recommendation: hint.action (mapped from Rust `action` field)
      - policyStrength: hint.policyStrength
      - riskScore: hint.riskScore
      - triggerCausalIds: hint.pattern_id ? [hint.pattern_id] : []

    The engine parameter is of type PolicyEngineImpl from @aura/contract.
    In test, use the mock engine pattern.
  </implementation>
</feature>
