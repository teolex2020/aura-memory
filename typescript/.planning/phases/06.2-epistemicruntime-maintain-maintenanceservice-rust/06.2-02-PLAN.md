<feature>
  <name>Concept surface function: filter, sort, limit, map disclosed concepts</name>
  <files>
    packages/concept/src/Surface.ts (NEW)
    packages/concept/src/Surface.test.ts (NEW)
  </files>
  <behavior>
    surfaceConcepts(engine, limit?):
      - Reads engine state via engine.stats()
      - Filters concepts where state is ConceptState.Stable or ConceptState.Candidate
      - Sorts by abstractionScore descending
      - Caps at limit (default 20, max no cap)
      - Maps each to SurfacedConcept: id, key, state, namespace, abstractionScore, beliefCount (belief_ids.length), recordCount (record_ids.length), coreTerms, recordIds
      - Returns ReadonlyArray&lt;SurfacedConcept&gt;
      - Empty state returns []

    surfaceConceptsFiltered(engine, limit?, namespace?):
      - Same as surfaceConcepts but additionally filters by exact namespace match
      - If namespace not provided: same as surfaceConcepts
      - Returns []
      - Note: Rust concept.rs surface filters from state.concepts which is the ConceptEngineState.concepts map (Record<string, ConceptCandidate>)

    Test cases:
      1. surfaceConcepts returns all eligible concepts sorted by score desc
      2. surfaceConcepts respects limit
      3. surfaceConceptsFiltered filters by namespace
      4. surfaceConcepts on empty engine returns []
      5. surfaceConceptsFiltered with non-existent namespace returns []
      6. surfaceConcepts excludes Rejected state concepts
      7. Multiple concepts with same score maintain deterministic order (secondary sort by key/id)
      8. surfaceConcepts default limit is 20 when no limit arg
  </behavior>
  <implementation>
    Import { Effect } from "effect"
    Import { SurfacedConcept } from "@aura/contract"
    Import type { ConceptEngineImpl, ConceptEngineState, ConceptCandidate } from "@aura/contract" (or from local engine module)

    The function signature:
    ```ts
    export function surfaceConcepts(
      engine: ConceptEngineImpl,
      limit?: number
    ): Effect.Effect<ReadonlyArray<SurfacedConcept>>
    ```

    Implementation:
    - yield* engine.stats() to get ConceptEngineState
    - Access state.concepts (Record<string, ConceptCandidate>)
    - Object.values(state.concepts) to get array
    - Filter: c.state === "Stable" || c.state === "Candidate" (use ConceptState enum)
    - Sort by abstraction_score desc, then by key asc for determinism
    - Apply max = limit ?? 20 (no upper cap per Rust)
    - Map to SurfacedConcept with camelCase field names

    surfaceConceptsFiltered adds Object.values(state.concepts).filter(c => c.namespace === namespace) before sorting.

    The engine parameter is of type ConceptEngineImpl from @aura/contract (the type that implements ConceptEngine's interface). In test, use the mock pattern.
  </implementation>
</feature>
