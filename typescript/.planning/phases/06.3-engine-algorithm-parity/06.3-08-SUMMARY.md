## Task Summary

### Task 1: 20+ field scoring — transition_lift, temporal_consistency, outcome_stability, causal_strength

**Commit:** `efdf956` — feat(06.3-08): implement score_pattern

**What was done:**

1. **Scoring constants** — Added Rust-aligned constants: scoring weights (W_TRANSITION_LIFT=0.35, W_TEMPORAL_CONSISTENCY=0.30, W_OUTCOME_STABILITY=0.20, W_SUPPORT=0.15), state thresholds (STABLE=0.75, CANDIDATE=0.50), MIN_SUPPORT=2, MAX_COUNTERFACTUAL_RATIO=0.50, EVIDENCE_WINDOW_SECS=86400, plus negative/positive outcome keyword lists.

2. **scorePattern function** — Pure function matching Rust score_pattern exactly:
   - **transition_lift**: P(effect|cause) = n / cause_count, P(effect) = effect_count / ns_total, raw_lift = min(P(e|c)/P(e), 5.0), normalized = raw_lift / 5.0
   - **temporal_consistency**: positive_gaps / total_pairs using cause_record_ids x effect_record_ids cross product (NOT edge-level gaps). Explicit floor at 0.60 for explicitly-supported patterns.
   - **outcome_stability**: 1 - cv where cv = sqrt(population_variance) / mean of effect record strengths. Neutral 0.5 when < 2 strengths or mean=0.
   - **support_score**: log2(n+1) / log2(21) — constant max_expected=20 (NOT dynamic max_support as the plan guessed, matching Rust exactly)
   - **causal_strength**: If n >= MIN_SUPPORT(2) or ExplicitTrusted bypass: full weighted sum (0.35*transition_lift + 0.30*temporal_consistency + 0.20*outcome_stability + 0.15*support_score). Otherwise: penalized transition_lift * 0.3.
   - **effect_record_signature_variants**: unique signatures from sorted, lowercased tags or semantic_type:content fallback
   - **positive/negative_effect_signals**: keyword matching on semantic_type "contradiction", record tags, and content against Rust's polarity keyword lists
   - **Legacy fields**: confidence = P(effect|cause), lift = raw_lift (unnormalized)

3. **Contract type additions** (Rule 2 deviation — needed for Rust parity):
   - temporal_support_count, explicit_support_total_for_cause, explicit_effect_variants_for_cause (populated in aggregateToPatterns)
   - effect_record_signature_variants, positive_effect_signals, negative_effect_signals (computed in scorePattern)

4. **TDD**: 12 RED tests (scorePattern not exported) → GREEN (all 12 pass). Tests cover: transition_lift with lift<5.0 and lift>5.0, temporal_consistency with positive and negative gaps, outcome_stability with multiple and single effects, support_score indirectly via strength, penalized causal_strength, full weighted combination, zero support edge case, legacy fields, polarity signals.

### Task 2: Evidence gates + corpus fingerprint + pattern state promotion

**Commit:** `e397aa5` — feat(06.3-08): implement evidence gates + corpus fingerprint + state promotion

**What was done:**

1. **Evidence gate functions** (all exported for testing):
   - **meetsSupportGate**: support_count >= 2 default, explicit_support_count >= 1 in ExplicitTrusted mode
   - **meetsRepeatedEvidenceGate**: Fast path (explicit >= 2 or temporal_windows >= 2 → true). StrictRepeatedWindows: false. TemporalClusterRecovery: temporal_support_count >= 2 AND explicit=0 AND counterevidence=0 AND signature_variants<=1 AND negative_signals=0 AND positive_signals>=1. ExplicitTrusted: explicit_support_count >= 1.
   - **meetsEvidenceGate**: support_gate AND repeated_evidence_gate
   - **meetsCounterfactualGate**: counterfactual_ratio (= counterevidence / (support+counterevidence)) > 0.50 → false. ExplicitTrusted bypass with mixed-polarity check. Otherwise: explicit_dominance (>70%) + effect_signature_consistency + effect_polarity_consistency gates.

2. **Corpus fingerprint** — `computeCorpusFingerprint(records)`:
   - Sorts record IDs, builds input string with: rid|namespace|created_at_bits|caused_by_id|causal_connections
   - created_at encoded as IEEE 754 double bits via Float64Array/BigInt64Array (exact float reproducibility)
   - Causal connections sorted for determinism
   - Uses xxhash-wasm (if initialized) or simpleHash64 fallback for synchronous test use

3. **Discover pipeline rewrite** — Full 5-phase pipeline:
   - **Phase 0**: Compute corpus fingerprint, skip if unchanged (preserves cached patterns)
   - **Phase 1**: extractEdges (from Plan 07)
   - **Phase 2**: belief_engine.stats() (from Plan 07)
   - **Phase 3**: aggregateToPatterns (from Plan 07)
   - **Phase 4**: Score each pattern (scorePattern), apply evidence gates (meetsEvidenceGate, meetsCounterfactualGate), classify state (Stable>=0.75, Candidate>=0.50, Rejected otherwise or gate failure)
   - Adds evidence gate stats to CausalReport (meeting_support_gate, meeting_repeated_window_gate, meeting_counterfactual_gate, blocked_by_evidence, blocked_by_counterfactual)
   - Populates avg_confidence and avg_lift from scored patterns

4. **TDD**: 20 RED tests (gates + fingerprint) → GREEN (all 20 pass). Tests cover: support gate pass/fail, ExplicitTrusted bypass, repeated evidence gate temporal_windows>=2 and explicit>=2 paths, ExplicitTrusted and StrictRepeatedWindows paths, counterfactual gate ratio pass/fail/boundary, state promotion Stable/Candidate/Rejected, fingerprint determinism across record IDs, namespaces, timestamps, caused_by_id, and causal connections.

### Formula Corrections vs Plan

The plan contained several formula guesses that did not match the Rust source. All were corrected to match Rust exactly:

**2. [Plan deviation] support_score formula: constant not dynamic**
- **Plan said:** ln(1+support)/ln(1+max_support) with dynamic max across all patterns
- **Rust actual:** log2(n+1) / log2(21) with constant max_expected=20
- **Resolution:** Used Rust formula. The plan's "NO AMBIGUITY" warning was self-contradicting.

**3. [Plan deviation] temporal_consistency formula: pairs, not edges**
- **Plan said:** positive_gap_edges / total edges
- **Rust actual:** positive_gaps / (cause_record_ids.len() * effect_record_ids.len()) — cross-product of cause×effect record pairs
- **Resolution:** Used Rust formula with record-pair cross product.

**4. [Plan deviation] outcome_stability formula: cv, not variance/mean**
- **Plan said:** outcome_stability = 1 - variance/mean
- **Rust actual:** outcome_stability = 1 - cv where cv = sqrt(population_variance) / mean
- **Resolution:** Used Rust formula with sqrt.

**5. [Plan deviation] confidence_score component does not exist in Rust**
- **Plan said:** Add confidence_score component (average of record confidences) to causal_strength
- **Rust actual:** score_pattern has exactly 4 components (transition_lift, temporal_consistency, outcome_stability, support_score)
- **Resolution:** Omitted confidence_score from causal_strength. Set legacy `confidence` field to P(effect|cause) for backward compatibility.

**6. [Plan deviation] counterfactual_ratio formula uses total, not support**
- **Plan said:** counterevidence / support
- **Rust actual:** counterevidence / (support_count + counterevidence)
- **Resolution:** Used Rust formula (total denominator).

## Known Stubs

1. **simpleHash64 fallback for corpus fingerprint** — When xxhash-wasm hasher is not yet initialized (synchronous context), `computeCorpusFingerprint` uses a simple non-cryptographic 64-bit hash. This is deterministic and produces distinct outputs for distinct inputs, but collision probability is higher than xxh3. In the discover pipeline, the hasher is typically initialized from the existing `_hasher` infrastructure. For tests, the simple hash is sufficient. **No immediate action needed — xxhash-wasm is already imported and initialized lazily.**
