## Decisions Made

- Used `as MaintenanceReport` cast instead of `satisfies MaintenanceReport` because `exactOptionalPropertyTypes: true` in tsconfig requires the exact return type for Effect.gen type inference
- Removed explicit return type annotation on runMaintenance to let TypeScript infer the Effect.gen return type (avoids incompatibility between the inferred requirements and the manually specified types)
- Imported MaintenanceService functions individually (runInitialPhases, buildSdrLookup, etc.) since there is no namespace export
- Imported contract types (MaintenanceReport, PhaseTimings, etc.) from @aura/contract, separate from function imports
