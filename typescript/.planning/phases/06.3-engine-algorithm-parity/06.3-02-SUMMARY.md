## Deviations from Plan

### Auto-Fixed Issues

**1. [Rule 1 - Bug] byteLength() required for content filter (UTF-16 ≠ UTF-8)**
- **Found during:** Task 1 GREEN testing — Chinese tests passed content filter with .length but failed coarse grouping assertions
- **Issue:** `rec.content.length` counts UTF-16 code units, not UTF-8 bytes. Rust's `content.len()` counts UTF-8 bytes. For Chinese text, a 9-char CJK string has .length=9 (UTF-16) but byteLength ≈ 27 (UTF-8). This caused records that should pass Rust's `< 10` byte guard to be incorrectly filtered.
- **Fix:** Implemented `byteLength(s)` using `new TextEncoder().encode(s).length` and used it in the content filter check. This matches Rust's byte-level content length behavior.
- **Files modified:** BeliefEngine.ts
- **Commit:** `57512e4`

**2. [Rule 1 - Bug] xxhash-wasm API mismatch (create64 vs h64)**
- **Found during:** Task 1 RED testing
- **Issue:** `xxhash().create64()` returns a streaming hasher with `update`/`digest` methods, not a one-shot `h64` method. This caused "hasher.h64 is not a function" errors.
- **Fix:** Changed to use `xxhash().then(h => ({ h64: (input: string) => hasher.h64(input) }))` matching the existing pattern in ConceptEngine.ts.
- **Files modified:** BeliefEngine.ts
- **Commit:** `57512e4`

**3. [Rule 1 - Bug] Test expectations updated for Rust-aligned subcluster behavior**
- **Found during:** Task 1 GREEN testing — Chinese tests expected old behavior (1 belief per coarse group with multiple hypotheses)
- **Issue:** Rust creates separate beliefs per SDR subcluster, not multiple hypotheses within one belief. Old tests assumed 3 beliefs for 3 coarse groups; Rust produces 6 beliefs (3 coarse × 2 SDR subclusters). Single-record subclusters skipped (Rust's `group_records.len() < 2` guard).
- **Fix:** Rewrote Chinese tests (BeliefEngine.zh.test.ts) to match Rust-aligned behavior: subclusters become separate beliefs, single-record subclusters produce 0 beliefs.
- **Files modified:** BeliefEngine.zh.test.ts
- **Commit:** `57512e4`

**4. [Rule 3 - Auto-fix] `coarse_groups` metric reports post-subcluster count**
- **Found during:** Task 1 GREEN testing
- **Issue:** BeliefReport.coarse_groups was `groups.size` (post-subcluster count) instead of `coarseGroups.size` (pre-subcluster count). The test expected 3 coarse groups but received 6.
- **Fix:** Changed report to use `coarseGroups.size` (pre-subcluster count).
- **Files modified:** BeliefEngine.ts
- **Commit:** `57512e4`
