## Task Summary

### Task 1: BridgeKey normalize + TagFamily backoff strategies (TDD)

**RED commit:** `af65b0d` ŌĆö 32 failing tests for normalizeBridgeTag, denseCorridorStableTags, denseBackoffGroupKey, and TagFamily integration.

**GREEN commit:** `2475a27` ŌĆö All functions implemented, update_with_sdr modified with TagFamily strategies.

**What was done:**

1. **normalizeBridgeTag** ŌĆö exported with optional `overrides?: Record<string, string>` parameter. Default bridge table from Rust (3 families, 6 input tags): "ui"/"frontend"ŌåÆ"frontend", "auth"/"authentication"ŌåÆ"authentication", "deploy"/"release"ŌåÆ"release". Overrides take priority over defaults.

2. **denseCorridorStableTags** ŌĆö counts tag frequency across records. Tags with count >= 4 OR is family tag are stable. Always includes family tag at position 0. Excludes generic tags (currently "alerts" per Rust).

3. **denseBackoffGroupKey** ŌĆö requires >= 4 records and >= 2 stable tags. Picks stable tags matching record's tags, truncates to 3. Formats as `namespace:tags:semantic_type`. Falls back to coarse_key if conditions not met.

4. **parseTagFamilyFromKey** ŌĆö extracts family tag (second segment) from coarse key.

5. **isGenericTagFamily** ŌĆö returns true for "alerts" (matching Rust).

6. **TagFamilyDenseBackoff pre-refinement** ŌĆö re-keys records with `denseBackoffGroupKey` before SDR subcluster when 4+ records and 2+ stable tags exist.

7. **TagFamilyAdaptive 2-pass** ŌĆö first pass with standard threshold (0.15). If all clusters are singletons and not generic family, retry with relaxed threshold (0.10).

8. **TagFamilyBackoff fallback** ŌĆö for Backoff/PairBackoff/DenseBackoff modes: when all clusters are singletons and not generic family, merge all records into broad bucket (single group under original coarse_key).

### Task 2: apply_layer_feedback rewrite with bounded clamping + BeliefFeedbackReport (TDD)

**RED commit:** `82a5acb` ŌĆö 7 failing tests for new apply_layer_feedback signature, clamping, stability, report fields, and volatility bounds.

**GREEN commit:** `4bfc7a0` ŌĆö apply_layer_feedback rewritten, BeliefFeedbackReport/FeedbackAuditEntry types defined, mock-based tests pass.

**What was done:**

1. **New signature:** `(causalEngine: CausalEngine.Interface, policyEngine: PolicyEngine.Interface) => Effect.Effect<BeliefFeedbackReport, never, EpistemicTrace>`. Uses Effect.gen for async stats() calls.

2. **Policy signal collection** ŌĆö iterates policy hints from `policyEngine.stats()`. Maps actionKind to direction: Avoid(-0.10), VerifyFirst(-0.06), Warn(-0.03), Prefer(+0.05), Recommend(+0.03). State-weighted: Stable(1.0), Candidate(0.6), Suppressed/Rejected skipped.

3. **Causal signal collection** ŌĆö iterates causal patterns from `causalEngine.stats()`. Maps state to deltas: Stable(+0.03 * strength), Candidate(+0.015 * strength), Rejected(-(0.02+0.04*pressure)), Invalidated skipped.

4. **Belief ID resolution** ŌĆö uses pattern's `cause_belief_id` and `effect_belief_id`. For policy hints, resolves through `pattern_id` ŌåÆ CausalPattern ŌåÆ belief IDs.

5. **Bounded clamping:**
   - Confidence delta: `Math.max(-0.18, Math.min(0.08, rawDelta))` matching Rust `MAX_TOTAL_FEEDBACK_DAMPING`/`MAX_TOTAL_FEEDBACK_BOOST`
   - Volatility delta: `Math.max(-0.06, Math.min(0.20, rawVolatilityDelta))` matching Rust `MAX_TOTAL_VOLATILITY_RELIEF`/`MAX_TOTAL_VOLATILITY_INCREASE`

6. **Stability preservation** ŌĆö stability is NOT modified; `stabilityBefore === stabilityAfter`. Score updated proportionally via `score * (newConfidence / oldConfidence)`.

7. **BeliefFeedbackReport** ŌĆö locally defined with 6 fields: `beliefsTouched`, `beliefsBoosted`, `beliefsDampened`, `netConfidenceDelta`, `netVolatilityDelta`, `entries`.

8. **FeedbackAuditEntry** ŌĆö 13 fields: `beliefId`, `sourceKind`, `sourceId`, `reason`, `deltaRequested`, `deltaApplied`, `confidenceBefore/After`, `volatilityBefore/After`, `volatilityDeltaApplied`, `stabilityBefore/After`, `stabilityDeltaApplied`. Proportional attribution via `appliedDelta * (entry.confidenceDelta / rawDelta)`.

9. **Imports** ŌĆö added `CausalEngine`, `CausalState`, `PolicyEngine`, `PolicyState`, `PolicyActionKind`, and related types from `@aura/contract`.
