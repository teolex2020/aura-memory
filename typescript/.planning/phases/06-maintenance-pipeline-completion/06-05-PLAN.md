    **EpistemicRuntimeImpl class:**

    The class has NO internal state of its own — all engine state is managed by the individual engines (BeliefEngineImpl, ConceptEngineImpl, etc.) which are obtained from the Effect context.

    `maintain(records: ReadonlyMap<string, Record>, sdr_lookup: SdrLookup): Effect.Effect<EpistemicReport, never, EpistemicTrace>`:
    ```typescript
    return Effect.gen(function* () {
      // Step 1: Get all engine services
      const beliefEngine = yield* Effect.service(BeliefEngine)
      const conceptEngine = yield* Effect.service(ConceptEngine)
      const causalEngine = yield* Effect.service(CausalEngine)
      const policyEngine = yield* Effect.service(PolicyEngine)

      // Step 2: Run pipeline sequentially
      // Belief -> Concept
      const beliefReport = yield* beliefEngine.update_with_sdr(records, sdr_lookup)
      const conceptReport = yield* conceptEngine.discover(beliefEngine, records, sdr_lookup)

      // Concept -> Causal (get concept state, pass to causal)
      const conceptState = yield* conceptEngine.stats()
      const causalReport = yield* causalEngine.discover(conceptState, records, sdr_lookup)

      // Causal -> Policy (get causal state, pass to policy)
      const causalState = yield* causalEngine.stats()
      const policyReport = yield* policyEngine.discover(causalState, records)

      // Step 3: Return consolidated report
      const report: EpistemicReport = {
        belief: beliefReport,
        concept: conceptReport,
        causal: causalReport,
        policy: policyReport
      }
      return report
    })
    ```

    **Getters** (use `Effect.service` to get each engine, then call `.stats()`):
    - `get_beliefs(): Effect.Effect<BeliefEngineState>` — yield* BeliefEngine then `.stats()`
    - `get_concepts(): Effect.Effect<ConceptEngineState>` — same pattern
    - `get_causal_patterns(): Effect.Effect<CausalEngineState>` — same pattern
    - `get_policy_hints(): Effect.Effect<PolicyEngineState>` — same pattern
    - `get_surfaced_concepts(...) and get_surfaced_policy_hints(...)` — keep as Effect.succeed([]) (not in Phase 6 scope)

    **EpistemicRuntimeLive:**
    ```typescript
    export const EpistemicRuntimeLive = Layer.succeed(EpistemicRuntime, new EpistemicRuntimeImpl())
    ```

    **CRITICAL:** The maintain() method uses engines via Effect.service(), meaning it depends on those Tags being provided via Layer. This is fine — the layers are composed in DefaultLayer.

    Each engine call MUST be `yield*` sequentially — the pipeline is inherently sequential (each engine depends on the previous engine's state).
