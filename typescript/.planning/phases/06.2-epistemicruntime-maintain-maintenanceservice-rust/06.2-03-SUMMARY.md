## Plan 06.2-03: Policy Surface Functions in @aura/policy

### What was built

Implemented `surfacePolicyHints` and `surfacePolicyHintsFiltered` Effect functions in `@aura/policy`. These filter disclosed policy hints (Stable/Candidate), sort by policyStrength desc with actionKind priority tiebreaker (Avoid > Warn > VerifyFirst > Recommend > Prefer), and return typed `SurfacedPolicyHint[]` results. Aligned with Rust `policy.rs` surface module per D-11, D-17.

### Tasks

1. **TDD RED** — Created `Surface.test.ts` with 12 tests covering filtering, sorting, actionKind tiebreaker, capping (max 10 global, max 3 per domain), deduplication, namespace filtering, and empty state handling.

2. **TDD GREEN** — Implemented `surfacePolicyHints(engine, limit?)` and `surfacePolicyHintsFiltered(engine, limit?, namespace?)` in `Surface.ts`. Filters Stable/Candidate (candidate requires confidence >= 0.55, strength >= 0.70), sorts by policyStrength desc → actionKind priority → Stable before Candidate → id, caps per-domain MAX_SURFACED_PER_DOMAIN=3 and global MAX_SURFACED_HINTS=10, deduplicates by key and recommendation text. All constants match Rust reference.
