## Task Summary

### Task 1: 4-dim policy strength scoring + recommendation templates

**RED commit:** `516591e` — `test(06.3-10): add failing tests for 4-dim policy strength scoring + recommendation templates (RED)`
**GREEN commit:** `ed07391` — `feat(06.3-10): implement 4-dim policy strength scoring + recommendation templates (GREEN)`

**What was done:**

1. **computePolicyStrength** — Pure function computing `0.35*causal_strength + 0.25*confidence + 0.20*utilityScore + 0.20*stability`, clamped to [0, 1]. All four weights verified against Rust policy.rs lines 34-37.

2. **generateRecommendation** — 5 deterministic templates matching Rust generate_recommendation (lines 694-719):
   - Avoid: `"Avoid: '{causeSummary}' in domain [{domain}] has been associated with negative outcomes."`
   - VerifyFirst: `"Verify first: '{causeSummary}' in domain [{domain}] has shown risk signals -- check before proceeding."`
   - Prefer: `"Prefer: '{causeSummary}' in domain [{domain}] has consistently led to positive outcomes."`
   - Recommend: `"Recommend: '{causeSummary}' in domain [{domain}] has shown positive signals."`
   - Warn: `"Warning: '{causeSummary}' in domain [{domain}] has a strong causal pattern but unclear polarity."`

3. **aggregateBeliefConfidence** — Record-level confidence average fallback with 0.50 neutral default when no belief/record data available.

4. **buildHints** — Full hint construction from PolicySeeds: domain extraction (top 2 cause record tags), cause summary truncation (80 chars), polarity classification, action mapping, 4-dim scoring, recommendation generation.

5. **discover integration** — Hints built and stored in engine state with state classification (Stable >= 0.75, Candidate >= 0.50). Post-suppression stats computed.

**TDD:** 8 RED tests (stubs return 0/"") → 8 GREEN (all scores/templates correct). Additional discover integration test validates hints_found > 0 and hint fields populated.

### Task 2: Suppression phase + surface alignment

**RED commit:** `7c6d9a3` — `test(06.3-10): add failing tests for suppression + surface alignment (RED)`
**GREEN commit:** `60cf964` — `feat(06.3-10): implement suppression phase + surface alignment (GREEN)`

**What was done:**

1. **applySuppression** — Conflict detection algorithm matching Rust apply_suppression (lines 727-786):
   - Pairs compared within same namespace+domain
   - Opposite polarity: one positive (Prefer/Recommend) vs one negative (Avoid/VerifyFirst)
   - Overlapping cause_record_ids (shared cause-side records)
   - Lower policy_strength hint gets state=Suppressed
   - Equal strength within 0.01: suppress neither (conservative)
   - Only Candidate/Stable hints considered (already Suppressed/Rejected excluded)
   - Wired into discover method after hint construction

2. **Surface.ts sort alignment** — Changed from actionKind-priority sort to Rust-aligned order:
   - Primary: policy_strength DESC
   - Secondary: confidence DESC
   - Tertiary: risk_score DESC
   - Tiebreak: Stable before Candidate
   - Final tiebreak: key ASC
   - Removed unused ACTION_KIND_PRIORITY constant

3. **cause_record_ids provenance** — Added `cause_record_ids: ReadonlyArray<string>` field to PolicyHint contract type. This field carries the CausalPattern's cause_record_ids into the hint for suppression overlap detection, matching Rust's hint.cause_record_ids.

**TDD:** 3 RED failures (2 suppression + 1 surface sort) → 3 GREEN (all tests pass). Updated existing surface sort test to match new Rust ordering.
